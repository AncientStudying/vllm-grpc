# Implementation Plan: Direct gRPC Client Library (Phase 4.2)

**Branch**: `008-direct-grpc-client` | **Date**: 2026-05-02 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/008-grpc-client-library/spec.md`

## Summary

Add a `packages/client` Python library that wraps the gRPC frontend with a clean async context-manager API, integrate it as a third benchmark target in the harness, extend the harness with a three-way comparison report, and add `py.typed` to the generated stubs package. Primary technical approach: persistent `grpc.aio.Channel` shared across concurrent requests; three-way orchestration in `bench_modal.py` reuses the same Modal deployment for both proxy and direct harness runs to avoid a second cold start.

## Technical Context

**Language/Version**: Python 3.11  
**Primary Dependencies**: `grpcio>=1.65`, `vllm-grpc-gen` (workspace), `mypy --strict`, `ruff`  
**Storage**: N/A (files written to `docs/benchmarks/`)  
**Testing**: `pytest` (unit), `mypy --strict` (type check), `ruff` (lint/format)  
**Target Platform**: Linux (Modal A10G for benchmark runs; macOS/Linux for library unit tests)  
**Project Type**: Python library (`packages/client`) + benchmark harness extension  
**Performance Goals**: Client library adds no measurable overhead beyond the gRPC call itself; three-way benchmark completes in under 45 minutes wall-clock  
**Constraints**: No TLS (plaintext gRPC, matching existing patterns); no retry logic in client; no streaming (deferred to Phase 5); no partial benchmark output files on failure  
**Scale/Scope**: One new package (`packages/client`); ~60 lines in `runner.py`; ~150 lines in `metrics.py`/`compare.py`/`reporter.py`; py.typed is one empty file

## Implementation Notes

### packages/client package scaffold

`packages/client/pyproject.toml` follows `packages/proxy/pyproject.toml` exactly:

```toml
[project]
name = "vllm-grpc-client"
version = "0.1.0"
dependencies = [
    "grpcio>=1.65",
    "vllm-grpc-gen",
]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[tool.hatch.build.targets.wheel]
packages = ["src/vllm_grpc_client"]
```

### Channel lifecycle

`VllmGrpcClient.__aenter__` calls `grpc.aio.insecure_channel(self.addr)`.  
`VllmGrpcClient.__aexit__` calls `await self._channel.close()`.  
`ChatClient` receives the channel reference — it does not own it.

### Type ignore removals in proxy/frontend

Only `# type: ignore[import-untyped]` comments that suppress errors from `vllm_grpc` imports are removed. All `# type: ignore[misc]` and `# type: ignore[type-arg]` comments (grpcio servicer inheritance) remain in place.

### bench_modal.py — gRPC phase ordering

The `serve_grpc_for_bench()` Modal function stays alive across both the proxy harness run and the gRPC-direct harness run. The local uvicorn proxy subprocess is spawned only for the proxy run and torn down before the direct run. The `grpc_addr` value from `modal.Dict` is used directly as the `VllmGrpcClient` address for the direct run.

### Failure semantics (FR-008)

If `run_grpc_target()` raises or produces any `error=True` results, `bench_modal.py` must not write any files to `docs/benchmarks/`. The existing two-way baseline files from Phase 4.1 are not overwritten.
